# 設定ファイルの階層と使い分け

## 階層構造

```
優先度（高い方が優先）:
1. Managed Policy（組織管理）     ← 変更不可
2. ユーザーレベル設定             ← tune で管理
3. プロジェクトレベル設定          ← tune で管理
```

## 各設定ファイルの役割

### CLAUDE.md

**役割**: プロジェクトやユーザーの方針・規約・技術スタックを宣言する。

| スコープ | パス | 用途 |
|---|---|---|
| ユーザー | `~/.claude/CLAUDE.md` | 全プロジェクト共通の方針 |
| プロジェクト | `<project>/CLAUDE.md` | プロジェクト固有の方針 |
| サブディレクトリ | `<project>/src/CLAUDE.md` | ディレクトリ固有の補足 |

**適している内容**:
- 技術スタックの宣言（言語、フレームワーク、パッケージマネージャー）
- コーディング規約（命名規則、ファイル構成）
- ワークフロー方針（コミット規約、ブランチ戦略）
- プロジェクト固有の制約や背景情報

**適していない内容**:
- 特定ファイルタイプに限定されるルール → rules/ を使う
- 複数ステップの手順 → skills/ を使う
- 一時的なタスク指示

### Rules（.claude/rules/*.md）

**役割**: 特定の条件下で適用される具体的なルールを定義する。

| スコープ | パス |
|---|---|
| ユーザー | `~/.claude/rules/*.md` |
| プロジェクト | `<project>/.claude/rules/*.md` |

**適している内容**:
- ファイルタイプ別のルール（`paths: "**/*.ts"` で TS ファイルにのみ適用）
- 具体的な Do/Don't（コード例付き）
- フレームワーク固有のパターン
- セキュリティ・品質に関する制約

**適していない内容**:
- プロジェクト全体の方針 → CLAUDE.md を使う
- 手順書 → skills/ を使う

**サイズガイドライン**:
- 理想: 100行未満
- 許容: 100-500行
- 500行超: 分割を検討

**フォーマット**: `claude-code-rules` スキルの references/rule-format.md を参照。

### Skills（.claude/skills/*/SKILL.md）

**役割**: 再利用可能なワークフロー・手順を定義する。

| スコープ | パス |
|---|---|
| ユーザー | `~/.claude/skills/*/SKILL.md` |
| プロジェクト | `<project>/.claude/skills/*/SKILL.md` |

**適している内容**:
- 複数ステップのワークフロー
- 外部ツールとの連携手順
- ドメイン固有の専門知識
- 再利用可能なスクリプトやテンプレート

**適していない内容**:
- 単純なルール → rules/ を使う
- プロジェクトの方針宣言 → CLAUDE.md を使う

## 決定フローチャート

```
フィードバックを受けた
│
├─ 特定のファイルタイプや状況に限定される？
│   ├─ Yes → rules/ に追加
│   └─ No ↓
│
├─ 複数ステップの手順やワークフロー？
│   ├─ Yes → skills/ に追加
│   └─ No ↓
│
├─ プロジェクト固有の方針？
│   ├─ Yes → プロジェクト CLAUDE.md に追加
│   └─ No ↓
│
└─ 全プロジェクト共通の方針
    └─ ユーザー ~/.claude/CLAUDE.md に追加
```

## スコープの判断

```
この設定は他のプロジェクトでも使う？
│
├─ Yes → ユーザーレベル（~/.claude/）
│   例: コミットメッセージは日本語、TDD を実践する
│
└─ No → プロジェクトレベル（<project>/.claude/）
    例: このプロジェクトでは pnpm を使う、API のベース URL
```

## アンチパターン

### 設定の重複
同じルールが CLAUDE.md と rules/ の両方に書かれている。
→ 一箇所にまとめ、もう一方は削除する。

### スコープの不一致
プロジェクト固有の設定がユーザーレベルに置かれている。
→ プロジェクトレベルに移動する。

### 巨大な CLAUDE.md
CLAUDE.md が肥大化して読みづらい。
→ 具体的なルールは rules/ に、手順は skills/ に分離する。

### ルールの矛盾
異なるファイルで矛盾するルールが定義されている。
→ 優先度を明確にするか、統合する。

### 過剰な規定
些細なことまでルール化して設定が膨大になっている。
→ Claude がデフォルトで適切に判断できることはルール化しない。
